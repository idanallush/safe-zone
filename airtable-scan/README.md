# Airtable System Scan — תיעוד מערכת לקלוד קוד

## מטרה
התיקייה הזו מכילה את כל המידע שקלוד קוד צריך כדי להבין את מערכת ה-Airtable שלך ולבנות משהו דומה כאפליקציה עצמאית.

## מה נמצא פה

### `/tables` — מבנה הנתונים
לכל טבלה ב-Airtable:
- קובץ CSV עם הנתונים (ייצוא מלא)
- צילום מסך של רשימת השדות (Fields view)
- קובץ טקסט עם תיאור הלוגיקה

### `/interfaces` — ממשקי המשתמש
צילומי מסך של כל Interface שבנית — גם במצב צפייה וגם במצב עריכה.

### `/views` — תצוגות מיוחדות
צילומי מסך של Views חשובים (פילטרים, groups, sorts) שמשפיעים על הלוגיקה.

### `/automations` — אוטומציות
צילומי מסך של כל אוטומציה — trigger + steps.

### `system-overview.md` — סקירה כללית
תיאור שלך במילים שלך על מה המערכת עושה.

---

## איך לסרוק — שלב אחרי שלב

### שלב 1: סקירה כללית
פתח את `system-overview.md` ומלא את הטמפלט. זה הדבר הכי חשוב — בלי זה קלוד קוד רואה נתונים בלי הקשר.

### שלב 2: טבלאות
לכל טבלה ב-Airtable:
1. **ייצא CSV** — לחץ על `...` > `Download CSV` ושמור ב-`/tables`
2. **צלם את השדות** — לחץ על `Fields` (בסרגל הצדדי), צלם מסך של כל השדות עם הטיפוסים שלהם. שמור כ-`[שם-טבלה]-fields.png`
3. **Formula / Rollup / Lookup** — אם יש שדות מחושבים, לחץ על כל אחד וצלם את הנוסחה. שמור כ-`[שם-טבלה]-formulas.png`

### שלב 3: ממשקים (Interfaces)
לכל Interface:
1. **צלם במצב רגיל** — ככה קלוד קוד רואה את ה-UX הסופי
2. **צלם במצב עריכה** — לחץ על Edit, צלם. ככה הוא רואה איזה אלמנטים, פילטרים, ו-data sources יש
3. שמור הכל ב-`/interfaces` עם שמות ברורים

### שלב 4: תצוגות (Views)
אם יש Views עם פילטרים/groups מיוחדים שמשפיעים על הלוגיקה:
1. צלם את ה-View עם הפילטר פתוח (כדי שיראה את התנאים)
2. שמור ב-`/views`

### שלב 5: אוטומציות
לכל Automation:
1. פתח אותה ב-Airtable
2. צלם את ה-Trigger (מה מפעיל אותה)
3. צלם כל Action/Step
4. שמור ב-`/automations`

## אחרי שהכל מוכן

תריץ את קלוד קוד עם הפקודה:

```
תקרא את כל מה שיש ב-reference/airtable-scan/ ותכתוב CLAUDE.md שמתעד את הארכיטקטורה המלאה של המערכת — טבלאות, קשרים, לוגיקה עסקית, וממשקים.
```

ואז כשתרצה לבנות:

```
בהתבסס על CLAUDE.md, תבנה לי אפליקציית ווב שמשחזרת את הפונקציונליות של המערכת הזו.
```
